# Milestones and Debt Cycles

This directory captures the design-first entry point for milestone work and
smaller debt-reduction cycles.

Use these notes to record:

- the tracked graph work item or intent that anchors the cycle
- the relevant sponsor actors
- the hill or operational outcome being improved
- the invariants that must not be violated
- the acceptance-test plan that will become executable spec
- the explicit non-goals for the cycle

These notes are intentionally smaller and more immediate than the core design
corpus in [`/Users/james/git/xyph/design`](../README.md). The core corpus
defines enduring product truth. The files here define the next bounded slice of
work.

Every new milestone or debt cycle should start here only after backlog
reconciliation has happened in the graph.
